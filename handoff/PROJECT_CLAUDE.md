## L1 수렴 기준 (mutable: true)

### P1 — 단일 에이전트 수렴
- 출력 변경량 < **5%** (문자 수준 diff 비율)
- **3회** 연속 임계값 이하일 때 자동 종료
- 비판가 맹점 보고 **3회** 연속 0건 시 수렴 인정

### P2 — L1 타임아웃
- 최대 반복 횟수: **20회** (무한 루프 방지)
- 타임아웃 시 현재 최선 출력으로 CP1 진입 + warn 로그 기록

---

## L2 수렴 기준 (mutable: true)

### P3 — 에이전트 간 수렴
- 에이전트 간 충돌(conflict) **0건**
- 통합 변경량 안정: 3회 연속 전체 delta < 3%
- 비판가의 크로스 피드백 맹점 3회 연속 0건

### P4 — L2 타임아웃
- 최대 크로스 피드백 라운드: **10회**
- 타임아웃 시 미해결 충돌 목록과 함께 CP2 진입

---

## 체크포인트 평가 기준 (공통)

CP1, CP2 모두 아래 4개 기준 전부 통과 시 `passed`:

| 기준 | 측정 방법 |
|------|-----------|
| 완결성 | 각 에이전트 출력 파일 존재 + 비어 있지 않음 |
| 일관성 | 에이전트 내부 자기모순 < 임계값 |
| 규칙 준수 | CLAUDE.md + PROJECT_CLAUDE.md 체크리스트 |
| 추적가능성 | 모든 결정에 T-{NNN} 티켓 ID 참조 존재 |

CP2 추가 기준:
- 충돌 해소 (conflict_free): 미해결 에이전트 간 모순 0건

---

## 티켓 규칙

- L1 티켓 전부 `done` → CP1 실행 가능
- L2 티켓 전부 `done` → CP2 실행 가능
- `critical` 우선순위 티켓 미해결 시 CP 진입 불가

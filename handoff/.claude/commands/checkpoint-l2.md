# /checkpoint-l2 — CP2 감독 명령어

## 트리거 조건
- 모든 L2 티켓 (level: L2) status == done
- 비판가 L2 수렴 신호 수신 (에이전트 간 충돌 0건 + 3회 연속)

## Claude Code 실행 절차

### 1. 출력 파일 수집
```
seasons/S01/tickets.json     → L2 완료 티켓 목록
seasons/S01/checkpoints.json → CP1 passed 확인, CP2 현재 상태
seasons/S01/logs.json        → L2 크로스 피드백 이력
```

### 2. 평가 체크리스트 (CP1 4요소 + CP2 추가 기준)
- [ ] **완결성**: L2 통합 출력 파일 존재하고 비어 있지 않음
- [ ] **일관성**: 에이전트 간 크로스 참조 모순 없음
- [ ] **규칙 준수**: CLAUDE.md + PROJECT_CLAUDE.md 전체 항목
- [ ] **추적가능성**: 모든 L2 티켓에 조율 근거 연결
- [ ] **충돌 해소 (CP2 전용)**: 비판가 보고 충돌 목록 전부 해소됨

### 3. 통합 제안서 작성 (통과 시)
CP2 통과 후 Claude Code는 통합 제안서를 생성한다.
이 제안서가 CP3 대시보드 미리보기 패널에 렌더링된다.

```markdown
## 통합 제안서 — S{NN} {프로젝트명}

### 전략가 최종 출력 요약
### 실행가 구현 계획 요약
### 비판가 최종 리스크 평가
### 에이전트 간 조율 결과
### 실행 권장 순서
```

### 4. 결과 처리

#### 통과 (passed)
```json
{
  "id": "CP2",
  "status": "passed",
  "timestamp": "<ISO8601>",
  "feedback": "<통합 제안서 요약>"
}
```
→ CP3 status: "waiting"으로 전환
→ 대시보드 CP3 승인 버튼 활성화

#### 실패 (failed)
→ 미해결 충돌 목록 명시
→ 해당 에이전트들 L2 재진입

## 주의사항
- CP1 passed 확인 필수. CP1 없이 CP2 실행 불가.
- 통합 제안서는 CP3 사용자 승인의 유일한 근거.
- 제안서 품질이 전체 시즌 결과물을 좌우함. 충분히 작성할 것.
